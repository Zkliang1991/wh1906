学习总结

经过了这个月的学习，让我对前端有了新的认识。这个阶段主要讲了nodejs和vue框架，node.js是使用使用express脚手架快速搭建一个项目。vue是使用vue-cli也能快速的搭建一个项目。Vue具有易用、灵活、高效等特点。 1、什么是MVC和MVVM MVC分为三块 ：model view 和controller MVVM分为三块：model view 和view-model 在vue中怎么实现数据的双向绑定，使用watch观察者模式监听对象数据的改变触发页面渲染 Obje.defineProrotype（） 2、虚拟dom 在js对象和真实dom树之间存在的一个许你对象，所有的dom树节点都是根据这个虚拟dom实现生成的 在虚拟dom向真实的dom树转换之前会根据diff算法动态的计算需要更改的标签，进行替换操作 虚拟dom优势： 可以针对不同的终端平台输出不同的页面展示节点比如： 网页、微信小程序、原生应用 在生成的时候只需要修改render方法渲染出不同的节点标签即可 浏览器生成dom树的时候非常消耗资源，因此引入虚拟dom概念 通过一定的算法优化之后能够非常快捷的根据数据生成真实的HTML节点 现在vue和react都是使用的虚拟dom 虚拟dom就是通过一个对象描述一个html结构： 通过js 写一个循环吧节点生成在页面上 需要修改某一个节点的值 可以直接修改对象上的属性 然后调用render方法重新渲染页面 渲染的时候会对虚拟dom中的几点作比对 只重新渲染 3、路由守卫 beforeEach表示每一个路由跳转之前执行，我们可以通过它判断用户是否登录 4、Vue项目的上线流程 1.使用脚手架搭建项目 2.写静态页面 3.调用服务器端接口版数据动态化 4.build上线 router-link、编程式跳转 5、Vuex 是一个全局状态管理工具，把所有的state数据存储在一棵单一的状态树上 使用mutations实现数据的改变 如果是异步操作需要dispatch派发一个action，然后在action中commit一个mutation改变数据 Vue的优点：简单，轻量，组件解耦，超快的虚拟dom元素操作，自底向上的渐进式的js框架，因为Vue还可以自定义组件，也可以自定义过滤器的使用规则。在生成自定义组件或自定义过滤器时，必须在生成Vue构造函数之前设定，否则会报错。因为之后定义不会被找到。而且vue也支持路由的使用，可以通过Vue-router在页面之间的跳转。通过路由功能，可以实现页面与页面之间的交互，以及当使用Axios http库时，可以实现利用JSON与后台之间进行的数据交互。 初次接触vue，vue给人的整体感觉就是非常的简明，使用起来条理非常清晰，感觉每一部分都是一个模块，用到哪一部分就写哪一块。很多方法都是使用函数封装的，使用的时候直接调用。 语法上来说，也是比较语义化的，相对其他框架，理解起来更加容易一些。虽然使用起来给人一种很简单的感觉，但是简单却不失优雅，功能上来说易于理解且功能完善，提供的API也是非常全面的。 框架大小上来说，也比较小巧，只有几十kb，使用时只需要引入即可，操作并不繁琐。大大的简化了代码量。Vue最大的特点就是数据驱动，当数据发生改变的时候，界面也会相应的变化，不需要再手动去更改DOM。vue在实例化的过程中，会遍历传给实例化对象选项中的data的所有属性，并使用Object.defineProperty(obj,key,{set:function(){},get:function(){}})把这些属性定义到实例上，当更改data里的数据时，会调用Object.defineProperty内部的set方法，在set方法中去通知视图更新，这样就实现了数据驱动。 另外vue的双向绑定也是个相对于其他框架更好的功能。每当 new 一个 Vue，主要做了两件事：第一个是监听数据：observe(data)，第二个是编译 HTML：nodeToFragement(id)。 在监听数据的过程中，会为 data 中的每一个属性生成一个主题对象 dep。在编译 HTML 的过程中，会为每个与数据绑定相关的节点生成一个监听者 watcher，watcher 会将自己添加到相应属性的 dep 中。这样就实现了 修改输入框内容 => 在事件回调函数中修改属性值 => 触发属性的 set 方法。
